# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within OpenCode2, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities
2. Email the maintainer directly or use GitHub's private vulnerability reporting
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- Acknowledgment of your report within 48 hours
- Regular updates on the progress
- Credit for the discovery (unless you prefer to remain anonymous)

## Security Best Practices

When using OpenCode2:

1. **API Keys**: Never commit API keys or secrets to the repository
2. **Permissions**: Review permission settings before running commands
3. **Local Models**: Prefer local models for sensitive code
4. **MCP Servers**: Only connect to trusted MCP servers
